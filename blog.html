var:
Scope: Function-scoped (not block-scoped). Hoisting: Variables declared with var are hoisted to the top of their function or global scope. Reassignment: Can be reassigned and redeclared within the same scope.

let:
Scope: Block-scoped. Hoisting: Variables declared with let are hoisted to the top of their block scope but are not initialized until the actual declaration. Reassignment: Can be reassigned but cannot be redeclared within the same scope.

const:
Scope: Block-scoped. Hoisting: Like let, const is hoisted to the top of its block scope but is not initialized until the actual declaration. Reassignment: Cannot be reassigned after declaration. However, for objects and arrays declared with const, their properties or elements can still be modified.

undefined:
Automatic Initialization: Variables that are declared but not assigned a value are automatically assigned the value undefined. Function Return: Functions without a return statement implicitly return undefined. Missing Function Arguments: Parameters that are not provided in a function call are undefined. Property Access: If you try to access an object property that doesn't exist, the result is undefined.
null:
Intentional Absence of Value: It is often used by developers to explicitly indicate that a variable or object property has no value or is intentionally empty. Object Initialization: It can be assigned to object properties or variables to indicate that they should not reference any object.

A REST API (Representational State Transfer Application Programming Interface) is a set of rules and conventions for building web services. It follows a stateless, client-server architecture with uniform interfaces, using standard HTTP methods for communication. Resources, identified by unique URIs, are manipulated through requests and responses, typically in JSON or XML format. RESTful APIs are widely used for building scalable and interoperable web applications.
